1. Introduction                                        *math-conceal-intro*

math-conceal.nvim provides math conceal for LaTeX, Typst, Markdown, and related
filetypes.

There are two rendering paths:

- ASCII/Unicode conceal: fast tree-sitter based symbol replacement using
  Neovim extmark conceal.
- Graphical equation conceal: terminal-graphics rendering for richer equation
  display, adapted from https://github.com/pxwg/typst-concealer, a fork of the
  original https://github.com/PartyWumpus/typst-concealer.

Graphical equation conceal supports:

- Typst math and code blocks.
- Markdown math through https://github.com/mitex-rs/mitex.

Graphical equation conceal uses the kitty graphics protocol. Use a compatible
terminal such as kitty or Ghostty.
The graphical path requires Neovim 0.11 or newer. ASCII/Unicode conceal does
not require the image-path APIs.

2. Setup                                                *math-conceal-setup*

Minimal setup:
>
  require("math-conceal").setup({
    conceal = { "greek", "script", "math", "font", "delim", "phy" },
    ft = { "plaintex", "tex", "context", "bibtex", "markdown", "typst" },
  })
<

lazy.nvim setup with graphical equation conceal:
>
  return {
    "pxwg/math-conceal.nvim",
    main = "math-conceal",
    build = "cargo build --release --manifest-path service/Cargo.toml",
    opts = {
      conceal = { "greek", "script", "math", "font", "delim", "phy" },
      ft = { "plaintex", "tex", "context", "bibtex", "markdown", "typst" },
      image = {
        enabled = true,
      },
    },
  }
<

The Rust service is required only when graphical equation rendering is enabled.
For rocks.nvim installs, install the optional service rock:
>
  :Rocks install math-conceal-service
<
LuaRocks uses a prebuilt service rock when one is available for your platform;
otherwise it builds the service from source and requires Rust/Cargo.

For source or lazy.nvim installs, build it manually or via a plugin manager
build hook:
>
  cargo build --release --manifest-path service/Cargo.toml
<

When a renderer `service_binary` is omitted, math-conceal checks:
>
  service/target/release/typst-concealer-service
  <rocks-tree>/bin/typst-concealer-service
  typst-concealer-service on $PATH
<

3. ASCII conceal                                      *math-conceal-ascii*

ASCII/Unicode conceal is enabled by the top-level options:
>
  require("math-conceal").setup({
    conceal = { "greek", "script", "math", "font", "delim", "phy" },
    ft = { "plaintex", "tex", "context", "bibtex", "markdown", "typst" },
    highlights = {
      ["@sup_symbol"] = { link = "@boolean" },
    },
  })
<

Options:

                                                        *math-conceal-conceal*
`conceal`~
        List of conceal query groups to enable. Built-in groups include:
        `greek`, `script`, `math`, `font`, `delim`, and `phy`.

                                                             *math-conceal-ft*
`ft`~
        Filetypes where ASCII/Unicode conceal is attached.

                                                     *math-conceal-highlights*
`highlights`~
        Highlight definitions for conceal capture groups.

                                                             *math-conceal-opt*
`opt`~
        Neovim window-local conceal options injected only into windows showing
        buffers where math-conceal is attached.
>
  require("math-conceal").setup({
    opt = {
      conceallevel = 2,
      concealcursor = "n",
    },
  })
<
        Defaults are `conceallevel=2` and `concealcursor=n`. This keeps conceal
        active in Normal mode while revealing source on the cursor line during
        Insert mode. Set `concealcursor="nci"` to keep the previous
        always-concealed cursor-line behavior, or use any valid Neovim
        'concealcursor' string. These options are restored when a window no
        longer shows a math-conceal-attached buffer.

                                                        *math-conceal-buffer*
`buffer`~
        Default buffer-local ASCII/Unicode conceal behavior.
>
  require("math-conceal").setup({
    buffer = {
      mode = "edit",
    },
  })
<

        `buffer.mode` accepts:

        - `edit`: expand only the concealed node under the cursor. This is the
          default.
        - `preview`: keep ASCII/Unicode conceal collapsed when the cursor moves
          over it.
        - `presentation`: keep plugin-managed conceal collapsed for rendered
          reading surfaces. Image-rendered rows are cursor-protected, live
          source reveal paths are disabled, and the buffer window is kept at
          the configured `opt` conceal settings. Visual selection temporarily
          reveals source so inline text ranges can be selected precisely.

        The same setting can be changed for one buffer:
>
  require("math-conceal").setup_buffer({ mode = "preview" })
  require("math-conceal").setup_buffer({ mode = "presentation" })
  require("math-conceal").setup_buffer(bufnr, { mode = "edit" })
<

                                                *math-conceal-attachment-api*
BUFFER ATTACHMENT API

`attach()` is the common buffer entry point for ASCII/Unicode and graphical
conceal. Normal filetype buffers use it through the bundled ftplugins. Virtual
or preview-buffer integrations can supply a logical source independently of
that buffer's concrete 'filetype' and name:
>
  local conceal = require("math-conceal")
  local attachment = conceal.attach(bufnr, {
    source = {
      kind = "markdown", -- latex, markdown, or typst
      filetype = "markdown",
      path = "/absolute/path/to/note.md",
    },
    surfaces = {
      unicode = true,
      image = true,
    },
    mode = "presentation",
  })

  attachment:detach()
<

`source.kind` selects one built-in logical source profile:

- `latex`: LaTeX root parser and ASCII/Unicode conceal.
- `markdown`: Markdown root parser, injected LaTeX conceal, and the configured
  Markdown image renderer.
- `typst`: Typst root parser and the configured Typst image renderer.

`source.filetype` is the logical source filetype exposed to renderer resolvers.
`source.path` is the real absolute source path used for project roots, imports,
preambles, and path filters. It should be supplied for anonymous preview
buffers. `surfaces.image=true` remains gated by `image.enabled`; native LaTeX
image rendering is not currently provided.

Every call returns an owner-qualified attachment handle. Multiple owners may
attach the same source. Detaching one handle preserves the other owners.
Passing the same `owner` to a later `attach()` replaces that owner's request;
the old handle then becomes stale and cannot detach the replacement.

Additional helpers:
>
  conceal.resolve_source(bufnr, source)
  conceal.get_attachment(bufnr)
  attachment:refresh()
  conceal.detach(attachment)
  conceal.detach(bufnr)
<

math-conceal explicitly obtains the source profile's Tree-sitter parser. It
does not own `vim.treesitter.start()` or `vim.treesitter.stop()`; editor and
preview hosts remain responsible for Tree-sitter highlighting lifecycle.

                                                     *math-conceal-snacks*
SNACKS PICKER PREVIEWS

When Snacks is available, math-conceal automatically wraps the stock
`Snacks.picker.preview.file` previewer. Typst, Markdown, and LaTeX file previews
receive ASCII/Unicode conceal. Typst and Markdown also request graphical
conceal when `image.enabled=true`.

Scratch preview buffers use presentation mode. If Snacks reuses an already
loaded source buffer, the adapter preserves its existing buffer mode and adds a
separate attachment owner, so closing the picker does not detach the ordinary
filetype owner.

Configuration:
>
  require("math-conceal").setup({
    integrations = {
      snacks = {
        enabled = true, -- default
        unicode = true,
        image = true,
        mode = "presentation",
      },
    },
  })
<

Set `integrations.snacks=false` to disable the default adapter.

Custom synchronous previewers can use the exported wrapper:
>
  local snacks_conceal = require("math-conceal.integrations.snacks")

  opts.preview = snacks_conceal.wrap(my_previewer, {
    source = function(ctx, bufnr, detected)
      return detected
    end,
  })
<

The source callback may change the detected `kind`, `filetype`, or `path`.
Custom asynchronous previewers should call `snacks_conceal.sync(ctx, opts)`
after committing preview content, then `snacks_conceal.detach(ctx)` during
cleanup. `resolve(ctx, opts)` is available when only source resolution is
needed.

4. Equation conceal                                      *math-conceal-image*

Graphical equation conceal is configured under the top-level `image` table:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
    },
  })
<

Renderer bindings are configured under `image.renderers`. The default bindings
enable Typst buffers through `renderers.typst` and Markdown buffers through
`renderers.markdown`.

Graphical equation conceal and ASCII conceal can be enabled together. ASCII
conceal still uses `ft`; graphical equation rendering uses
`image.renderers.<name>.filetypes`.

Default renderer bindings:

- `typst`: Typst math and supported top-level Typst code blocks.
- `markdown`: `$...$`, `$$...$$`, `\(...\)`, and `\[...\]`
  Markdown math rendered through MiTeX on the Typst service backend.

5. Equation conceal examples                    *math-conceal-image-examples*

Typst only:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
      renderers = {
        typst = {
          filetypes = { "typst" },
        },
        markdown = {
          filetypes = {},
        },
      },
    },
  })
<

Typst and Markdown:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
    },
  })
<

Typst and plugin-owned Markdown-like buffers:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
      renderers = {
        markdown = {
          filetypes = { "markdown", "copilot-chat" },
        },
      },
    },
  })
<

Project-aware Typst rendering:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
      renderers = {
        typst = {
          filetypes = { "typst" },
          root = function(ctx)
            local wiki = vim.fs.normalize(vim.fn.expand("~/wiki"))
            local path = vim.fs.normalize(ctx.path)
            if path == wiki or path:sub(1, #wiki + 1) == wiki .. "/" then
              return wiki
            end
          end,
          inputs = { "concealed=true", "preview=true" },
          preamble_file = function(ctx)
            if ctx.path:match("/wiki/") then
              return vim.fn.expand("~/wiki/concealer-context.typ")
            end
          end,
        },
      },
    },
  })
<

Restrict rendering by path:
>
  require("math-conceal").setup({
    image = {
      enabled = true,
      renderers = {
        typst = {
          render_paths = {
            exclude = { "/vendor/", "/node_modules/", "/large%-generated/" },
          },
        },
      },
    },
  })
<

6. Equation conceal options                      *math-conceal-image-options*

                                                         *math-conceal-image.enabled*
`image.enabled`~
        Boolean. Enables graphical equation conceal. Default: false.

                                                       *math-conceal-image.renderers*
`image.renderers`~
        Table of renderer bindings. Defaults include:
>
        renderers = {
          typst = {
            filetypes = { "typst" },
          },
          markdown = {
            filetypes = { "markdown" },
            mitex_package = "@preview/mitex:0.2.7",
          },
        }
<

                                        *math-conceal-image.renderers.filetypes*
`image.renderers.<name>.filetypes`~
        Concrete Neovim filetypes managed by one renderer binding.

                                  *math-conceal-image.renderers.service_binary*
`image.renderers.<name>.service_binary`~
        Path to the Rust compiler service for one renderer binding. If omitted,
        math-conceal tries:
>
        service/target/release/typst-concealer-service
        <rocks-tree>/bin/typst-concealer-service
        typst-concealer-service on $PATH
<

                                              *math-conceal-image.enabled_by_default*
`image.enabled_by_default`~
        Boolean. Enables graphical equation conceal automatically for supported
        buffers. Default: true inside the renderer.

                                                   *math-conceal-image.styling_type*
`image.styling_type`~
        One of `colorscheme`, `simple`, or `none`.

        `colorscheme` matches the current Normal foreground and uses a
        transparent page fill.

        `simple` uses a simple document prelude without colorscheme matching.
        For both modes, render slots re-assert their auto-sized page setup
        after collected Typst prelude units, so document page settings such as
        `#set page(paper: "a4")` do not force formula images to page size.

        `none` does not inject styling. Use this when your project prelude fully
        controls the rendered output.

                                                         *math-conceal-image.color*
`image.color`~
        Typst color string used when `styling_type = "colorscheme"`. When nil,
        math-conceal derives the color from the current `Normal` highlight.

                                                           *math-conceal-image.ppi*
`image.ppi`~
        Fallback render PPI when terminal cell pixel size cannot be detected.
        Default: 300.

                                              *math-conceal-image.math_baseline_pt*
`image.math_baseline_pt`~
        Baseline point size used to derive render PPI from terminal cell height.
        Default: 11.

                                              *math-conceal-image.live_preview_enabled*
`image.live_preview_enabled`~
        Boolean. Enables the cursor-following live preview in normal, insert,
        and visual modes.
        Default: true.

                                           *math-conceal-image.preview_idle_timeout_ms*
`image.preview_idle_timeout_ms`~
        Milliseconds before an idle live-preview service is stopped after the
        preview surface is cleared. Set to a negative value to keep the service
        alive. Default: 1000.

                                               *math-conceal-image.hidden_service_idle_ms*
`image.hidden_service_idle_ms`~
        Milliseconds before services for a hidden buffer are stopped. The live
        preview service may be stopped immediately at timeout; the full render
        service is stopped only when it has no active or queued request. Set to
        a negative value to keep hidden-buffer services alive. Default: 2000.

                                      *math-conceal-image.renderers.live_debounce*
`image.renderers.<name>.live_debounce`~
        Delay in milliseconds for text-change live preview updates for a
        renderer. Cursor movement preview is immediate. Typst defaults to 0.

                                        *math-conceal-image.cursor_hover_throttle_ms*
`image.cursor_hover_throttle_ms`~
        Throttle delay for cursor hover synchronization. Default: 0.

                                                *math-conceal-image.conceal_in_normal*
`image.conceal_in_normal`~
        Boolean. Keeps concealing when the cursor is on a rendered item in
        normal mode. Default: false.

                                           *math-conceal-image.renderers.header*
`image.renderers.<name>.header`~
        Renderer-scoped Typst code prepended to generated render documents.

                                  *math-conceal-image.renderers.typst.code_render.allow*
`image.renderers.typst.code_render.allow`~
        List or map of global user Typst code names allowed to enter the
        renderer in addition to math-conceal's built-in predictable Typst code
        allowlist. Structural code such as `#import`, `#let`, and `#set`
        remains context-only. Selector show rules such as
        `#show math.equation: ...` remain context-only; bare document
        transformers such as `#show: conf.with(...)` are not replayed into
        isolated render preludes.
>
        image = {
          renderers = {
            typst = {
              code_render = {
                allow = { "theorem", "lemma", "remark" },
              },
            },
          },
        }
<

                                             *math-conceal-image.renderers.root*
`image.renderers.<name>.root`~
        String or function returning the Typst root directory passed to the
        compiler service. Function form receives one context table:
        `{ bufnr, kind, filetype, path, cwd }`.

                                           *math-conceal-image.renderers.inputs*
`image.renderers.<name>.inputs`~
        Table, list of `key=value` strings, or function returning Typst input
        values for one renderer binding.

                                    *math-conceal-image.renderers.preamble_file*
`image.renderers.<name>.preamble_file`~
        String or function returning an absolute `.typ` file to include at the
        top of generated render documents for one renderer binding.

                                      *math-conceal-image.renderers.render_paths*
`image.renderers.<name>.render_paths`~
        Optional path exclude rules. String rules are Lua patterns matched
        against normalized absolute paths. Function rules receive the renderer
        attach context and should return true when matched.
>
        image = {
          renderers = {
            typst = {
              render_paths = {
                exclude = {
                  "/vendor/",
                  function(ctx)
                    return ctx.path:match("/generated/") ~= nil
                  end,
                },
              },
            },
          },
        }
<

                                     *math-conceal-image.renderers.mitex_package*
`image.renderers.markdown.mitex_package`~
        Typst package spec used for Markdown LaTeX math conversion through
        MiTeX. Default: `@preview/mitex:0.2.7`.

7. Equation conceal API                              *math-conceal-image-api*

The public renderer module is:
>
  local image = require("math-conceal.image")
<

Available buffer controls:
>
  image.enable_buf(bufnr)
  image.disable_buf(bufnr)
  image.toggle_buf(bufnr)
  image.rerender_buf(bufnr)
<

These are useful for keymaps:
>
  vim.keymap.set("n", "<leader>mi", function()
    require("math-conceal.image").toggle_buf(0)
  end, { desc = "Toggle graphical equation conceal" })
<

Query helpers:
>
  image.source_kind_for_bufnr(bufnr)
  image.is_supported_bufnr(bufnr)
  image.is_render_allowed(bufnr)
<

Source-aware programmatic integrations should prefer the unified top-level
`attach()` API described at |math-conceal-attachment-api| so ASCII/Unicode and
image surfaces share one source descriptor and lifecycle. The image module
controls remain available for image-only user commands and diagnostics:
>
  local image = require("math-conceal.image")
  if image.is_supported_bufnr(bufnr) and image.is_render_allowed(bufnr) then
    image.enable_buf(bufnr)
  end
  image.rerender_buf(bufnr)
<

8. Troubleshooting                            *math-conceal-troubleshooting*

Run the built-in health checks first:
>
        :checkhealth math-conceal
<
They report Neovim/API compatibility, window-scoped placement support,
terminal graphics support, realization adapters, and render service binaries.

No rendered equations appear~
        Check terminal support first. Graphical equation conceal requires kitty
        graphics protocol support.

        Confirm the service is installed or built:
>
        :Rocks install math-conceal-service
        test -x service/target/release/typst-concealer-service
        command -v typst-concealer-service
<

        Confirm graphical equation conceal is enabled:
>
        :lua print(vim.inspect(require("math-conceal").opts.image))
<

Service cannot be spawned~
        Set the renderer service binary to an absolute path:
>
        image = {
          renderers = {
            typst = {
              service_binary = "/absolute/path/to/typst-concealer-service",
            },
            markdown = {
              service_binary = "/absolute/path/to/typst-concealer-service",
            },
          },
        }
<

Markdown math does not render~
        Ensure Markdown is enabled for graphical equation conceal:
>
        image = {
          enabled = true,
          renderers = {
            markdown = {
              filetypes = { "markdown" },
            },
          },
        }
<

Custom Markdown-like filetypes do not render~
        Add the concrete filetype to the Markdown renderer:
>
        image = {
          renderers = {
            markdown = {
              filetypes = { "markdown", "copilot-chat" },
            },
          },
        }
<

Project imports fail~
        Configure renderer-scoped `root`, `inputs`, or `preamble_file` so
        snippets compile under the same context as the real project.
